# Shadowverse EVOLVE 総合ルール コア要約
# ver 1.24.1 準拠 (2026/01/29)
# ※ CLI Agent (Claude Code / Gemini CLI / Codex) システムプロンプト向け圧縮版

---

## 1. ゲーム概要

- 2人対戦カードゲーム
- **勝利条件**: 相手リーダーの体力を0以下にする / 相手デッキ0枚時にドローさせる
- **敗北条件**: 自リーダー体力0以下 / デッキ0枚でドロー / 投了宣言
- **引き分け**: 両者同時敗北
- **大原則**: カードテキストはルールに優先する。実行不可能な指示は無視する。禁止効果は常に優先。

## 2. カード情報

### カード種類
| 種類 | 英語表記 | 特殊種類 |
|------|----------|----------|
| リーダー | Leader | - |
| フォロワー | Follower | EVOLVE, ADVANCE |
| アミュレット | Amulet | - |
| スペル | Spell | - |

### 主要情報
- **カード名**: 固有名称。『名称』表記はカード名の完全一致または部分一致
- **クラス**: ニュートラル/エルフ/ロイヤル/ウィッチ/ドラゴン/ナイトメア/ビショップ
- **タイプ**: ルール上の意味なし。他カードから参照される特徴語
- **コスト**: プレイに必要なPP数
- **攻撃力**: 攻撃時に与えるダメージ基準値
- **体力**: 破壊に必要なダメージ数。リーダーは初期20
- **テキスト**: 能力・効果の記述。()内の注釈文はゲームに影響しない

## 3. プレイヤー情報

### PP/EP/SEP
| 値 | 下限 | 上限 | 用途 |
|----|------|------|------|
| PP最大値 | 0 | 10 | ターン開始時にPPをこの値に設定 |
| PP | 0 | PP最大値 | カードプレイ・進化のコスト |
| EP | 0 | なし | 進化コストのPP1点をEP1点で代替可 |
| SEP | 0 | なし | 超進化に消費（初期値1） |

### 初期値
- 先攻EP: 0 / 後攻EP: 3
- SEP: 各1
- リーダー体力: 各20
- PP・PP最大値: 各0

### 経過ターン数
- 各プレイヤーがターンプレイヤーとしてスタートフェイズを開始した回数

## 5. ターン構造

```
┌─ スタートフェイズ ─────────────────────────┐
│ 1. PP最大値+1 (10未満なら)                    │
│ 2. PP = PP最大値に回復                         │
│ 3. 自場カード全スタンド                        │
│ 4. 1枚ドロー (先攻1ターン目は不可)            │
│ 5. チェックタイミング                          │
└──────────────────────────────────────────────┘
         ↓
┌─ メインフェイズ ──────────────────────────┐
│ 以下を繰り返し選択:                            │
│  • カードプレイ (手札/EXエリアから)            │
│  • 起動能力プレイ                              │
│  • フォロワーで攻撃                            │
│  • メインフェイズ終了 → エンドフェイズへ      │
│ ※各行動後にチェックタイミング発生             │
└──────────────────────────────────────────────┘
         ↓
┌─ エンドフェイズ ──────────────────────────┐
│ 1. エンドフェイズ誘発                          │
│ 2. チェックタイミング                          │
│ 3. 守護持ちを望む数アクト                      │
│ 4. チェックタイミング                          │
│ 5. 非ターンPがクイック使用可                   │
│ 6. 手札上限超過→捨てる                        │
│ 7. 「ターン終わりまで」効果除去                │
│ 8. ターン終了→相手のターンへ                  │
└──────────────────────────────────────────────┘
```

## 6. メインフェイズ詳細

### カードプレイ手順
1. カードを指定・公開し解決領域へ移動
2. 前提選択肢の決定（チョイス等）
3. 目標（「選ぶ」対象）の選択
4. 値の割り振り決定
5. コスト決定・支払い
6. 場の上限確認（フォロワー/アミュレット）
7. 「プレイされた」事象発生
8. 解決（場に出す or 効果実行）
9. 残存カード→墓場 / 能力→除去

### 攻撃手順
1. **攻撃フォロワー選択**: 自分のスタンド状態フォロワー
   - 条件: ターン最初から連続マスター or このターン進化済
2. **攻撃目標選択**:
   - 相手アクト状態フォロワー (常に可)
   - 相手リーダー (ターン最初から連続マスターの場合のみ)
   - 守護持ちアクトフォロワーがいれば優先選択
3. 攻撃フォロワーをアクト
4. 「攻撃した」事象発生（目標がフォロワーなら交戦状態）
5. チェックタイミング
6. 非ターンPのクイック使用機会
7. **ダメージ交換**: 攻撃フォロワー攻撃力→目標 / 目標攻撃力→攻撃フォロワー（同時）
8. チェックタイミング → 攻撃終了

## 7. 進化システム

### 通常進化
- **起動能力**として実行（1ターン1回制限）
- **コスト**: エボルヴデッキから適正カード1枚公開 + PP（PP1点をEP1点で代替可）
- **処理**: エボルヴカードを進化領域に置き、場のカードに関連付け
- **効果**: コスト以外の情報が進化後カードの情報に変更。攻撃力/体力の差分は引き継ぎ
- 進化前の能力は失われ、進化後の能力を得る
- スタンド/アクト状態は変わらない

### 超進化
- **条件**: 経過ターン数が先攻7以上/後攻6以上
- **追加コスト**: SEP 1点
- **追加効果**: 攻撃力+1/体力+1、「超進化した」事象発生

## 8. 能力の分類

| 種類 | 表記例 | 説明 |
|------|--------|------|
| 起動能力 | ◆(コスト):(効果) | プレイヤーが能動的に使用 |
| 自動能力 | (条件)とき、(効果) | 誘発条件で自動プレイ（強制） |
| 永続能力 | (上記以外) | 常時効果発生 |
| スペル能力 | スペルのテキスト | スペル固有 |

### 効果の分類
- **単発効果**: 1回実行して終了
- **継続効果**: 一定期間有効（「このゲーム中」含む）
- **置換効果**: 事象Aの代わりに事象Bを実行（「～する代わりに」表記）

---

## 9. チェックタイミング

1. ルール処理を全て同時実行（繰り返し）
2. ターンPの待機自動能力を1つプレイ・解決 → 1に戻る
3. 非ターンPの待機自動能力を1つプレイ・解決 → 1に戻る
4. 終了

### 主要ルール処理
- リーダー体力0以下 → 敗北
- デッキ0枚ドロー → 敗北
- フォロワー体力0以下 → 破壊（墓場へ）
- 必殺と交戦したフォロワー → 破壊
- 場の枚数超過 → 超過分を墓場へ

## 10. 主要キーワード能力

| キーワード | 種類 | 効果 |
|-----------|------|------|
| 守護 | 永続 | 場に出す際アクト可。相手はアクト守護を優先攻撃。エンドフェイズにアクト可 |
| 疾走 | 永続 | 場に出たターンからリーダー含め攻撃可能 |
| 突進 | 永続 | 場に出たターンからアクト状態フォロワーのみ攻撃可能 |
| 必殺 | 永続 | 交戦フォロワーを次ルール処理で破壊（ダメージ0でも有効） |
| ドレイン | 自動 | 攻撃ダメージ付与時、リーダー体力をその値回復（複数持ちは単一扱い） |
| オーラ | 永続 | 相手のカード/能力で選ばれない（攻撃目標には選択可） |
| 威圧 | 永続 | 相手の攻撃目標にならない |
| 指定攻撃 | 永続 | スタンド状態フォロワーも攻撃目標に選択可 |
| クイック | 永続 | 相手ターンの特定タイミング(攻撃後/エンド時)でプレイ可。自ターンメインでも可 |
| ファンファーレ | 自動 | 場以外から場に出たとき誘発 |
| ラストワード | 自動 | 場から墓場に置かれたとき誘発 |
| 進化時 | 自動 | 進化したとき誘発 |
| 攻撃時 | 自動 | 攻撃したとき誘発 |
| 超進化時 | 自動 | 超進化したとき誘発 |
| アドバンス起動 | 起動 | PP1点をEP1点で代替可。進化相当能力扱い |

## 11. デッキ構築ルール

- **リーダーカード**: 1枚
- **メインデッキ**: 40～50枚（リーダー/エボルヴ/アドバンス/トークン不可）
- **エボルヴデッキ**: 0～10枚（エボルヴ/アドバンスのみ）
- **同名制限**: メイン3枚+エボルヴ3枚=計6枚まで
- **クラス基準**: 全カードがリーダーと同クラスかニュートラル
- **タイトル基準**: 全カードが同一タイトル（リーダー含む）

## 12. クラス別キーワード

| クラス | キーワード | 種別 | 意味 |
|--------|-----------|------|------|
| エルフ | コンボ N | 特定表記 | このターンN枚以上プレイ済（トークン含む全領域） |
| ウィッチ | スペルチェイン/SC N | 特定表記 | 自墓場スペルN枚以上（解決中の自身は含まない。解決開始時固定） |
| ウィッチ | スタック | 永続+起動 | 場に出る時カウンター1個付与。場離れ時カウンター1個除去で残る。カウンター0→ルール処理で墓場 |
| ウィッチ | 土の秘術 N | 永続 | プレイ時追加コストとしてスタックカウンターN個除去。除去後0個なら墓場行き |
| ドラゴン | 覚醒 | 特定表記 | 自PP最大値7以上 |
| ナイトメア | ネクロチャージ/NC N | 特定表記 | 自墓場カードN枚以上（解決中の自身は含まない。解決開始時固定） |
| ナイトメア | 真紅 | 特定表記 | 自ターン中にリーダー体力が減少したことがある |

## 13. タイトル別キーワード

### 13.1 ウマ娘 プリティーダービー

**食事 (特定表記)**
- 「N回食事する」: カードに関連付けされた出走領域のカードがない場合、エボルヴデッキの『にんじん』N枚を出走領域に置き関連付け
- すでに関連付けカードが出走領域にある場合は何もしない（コスト不可）
- 場を離れると出走領域との関連付けを失う

**食事能力 (キーワード)**
- 食事処理を含む起動能力。PP1点をEP1点で代替可。**進化相当能力**

**出走 (特定表記)**
- 「N回出走する」: 出走領域に関連付けカードがある限り【突進】を持つ。「出走した」事象N回発生

**出走時 (キーワード)**
- カードが出走したときに誘発する自動能力

### 13.2 アイドルマスター シンデレラガールズ

**魔法のアイテム**
- このタイトル基準デッキの場合、先攻決定直前にEXエリアに『魔法のアイテム』トークン5つを配置

**レッスン N (特定表記)**
- EXエリアの『魔法のアイテム』をN枚消滅させる

### 13.3 カードファイト!! ヴァンガード

**トリガーアイコン**
- カード右上のアイコン。4種: クリティカル/ドロー/スタンド/ヒール

**デッキ構築追加条件**
- 【スタートアミュレット】持ちカード1枚以上必須
- トリガー持ちカードのクラスはリーダーと一致必須

**ゲーム開始手順変更**
- デッキ配置前に【スタートアミュレット】持ちカード1枚を裏向きで場に出す
- 引き直し後に表向きにする

**スタートアミュレット (キーワード)**
- デッキ構築制限永続能力。このカードと異なるカード名の【スタートアミュレット】は入れられない

**ドライブチェック (特定表記)**
- デッキ上1枚→トリガー領域。トリガーアイコンがあれば対応処理を任意実行:
  - クリティカル: 自フォロワー1体の攻撃力+2
  - ドロー: 1枚引く
  - スタンド: 自フォロワー1体スタンド（そのターン相手リーダー攻撃不可）
  - ヒール: 自リーダー体力+3
- トリガー処理実行→カード墓場 / トリガーなしor不実行→デッキ下

**シングルドライブ / ツインドライブ (キーワード)**
- 【攻撃時】ドライブチェック1回 / 2回連続実行

**ドライブ (キーワード)**
- フォロワーが得る永続能力。取得時に【シングルドライブ】と【突進】も同時取得
- 一度ドライブを持ったことがあれば再取得不可

**ドライブ獲得時 (キーワード)**
- ドライブを得たときに誘発する自動能力

**憑依/ライド (キーワード)**
- フォロワーにドライブを持たせる起動能力
- コスト: エボルヴデッキの『ドライブポイント』1枚をドライブ領域に置き関連付け + PP（PP1点をEP1点で代替可）
- このゲーム中1回のみ。**進化相当能力**
- 場を離れるとドライブ領域との関連付けを失う

### 13.4 プリンセスコネクト！Re:Dive

**ユニオンバースト (キーワード)**
- 「ユニオンバースト （能力）」形式。「UB」表記あり
- **有効条件**: タイトル基準で「プリンセスコネクト！Re:Dive」デッキの場合のみ有効。クラス基準や他タイトル基準では常に無効
- 「発動した」= 適正にプレイされ解決された
- 「元のコストを支払わずに発動する」= コスト0としてプレイ。X含みならX=0

**イクイップメント (種類) / 装備 (特定表記)**
- トークンとして作成。イクイップメント領域でのみ存在可能
- 「フォロワーは名称を装備する」= トークン作成→関連付け→「装備した」誘発
- フォロワーは複数装備可。場を離れると関連付け喪失

## 14. 特定表記クイックリファレンス

- **破壊する**: 場→オーナー墓場
- **消滅する**: 任意領域→オーナー消滅領域
- **探す**: デッキ確認→カード取得→シャッフル
- **引く**: デッキ上→手札（デッキ0枚なら敗北）
- **捨てる**: 手札→自分墓場
- **変身する**: 消滅→同数のトークン作成
- **進化する**: エボルヴカードを進化領域に関連付け
- **ダメージ**: 体力を数値分減少（負の値になりうる）
- **回復する(PP)**: PPに値を加算（最大値超過不可）
- **奪う/渡す**: マスター変更。場→場移動。オーナーは不変
- **チョイスする**: 選択肢から指定数選んで実行（プレイ時決定）
- **X**: テキスト内で決定 or プレイ時にプレイヤーが決定
- **～につき**: 単位数×数値の合計を1回で実行（端数切捨）
- **半分**: 奇数なら切り上げ
- **箱詰め**: 能力喪失+スタートフェイズのルールスタンド不可（効果によるスタンドは可）
- **操縦する**: 攻撃力・体力を持つアミュレットをそのターン中フォロワーにする

---

## 15. トークン基本ルール

- カードではないがカードとして扱う
- **フォロワー/アミュレットトークン**: EXエリア・場・解決領域でのみ存在可
- **スペルトークン**: EXエリア・解決領域でのみ存在可
- **イクイップメントトークン**: イクイップメント領域でのみ存在可
- 存在不可の領域に移動→即座に消去（チェックタイミング不要）
- オーナー/マスター = 作成された領域のプレイヤー
