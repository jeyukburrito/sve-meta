# SVE 메타 분석 — 워크플로 스킬 목록

Claude에게 아래 문장을 그대로 말하면 해당 파이프라인이 실행됩니다.

---

## 📥 데이터 수집

| 명령 | 동작 |
|------|------|
| `최근 N개 대회 수집해줘` | bushi-navi 개인전 최근 N개 대회 결과 수집 → `data/개인전_v2.txt` 추가 |
| `트리오 최근 N개 대회 수집해줘` | 트리오 대회 수집 → `data/트리오.txt` 추가 |
| `대회코드 {id} 수집해줘` | 특정 대회 1건만 수집 |
| `{날짜} 이후 대회 수집해줘` | 날짜 필터 수집 |

**수집 파이프라인**
1. Playwright → bushi-navi API 인터셉트 → 순위/클래스/덱코드 추출
2. decklog 방문 → `img[title]` 카드 코드 수집
3. `normalize_deck()` → canonical 코드 변환
4. `data/개인전_v2.txt` 에 TSV 행 추가

---

## 🃏 카드 DB 수집

| 명령 | 동작 |
|------|------|
| `카드DB {세트코드} 수집해줘` | 예: `카드DB BP18 수집해줘` |
| `카드DB 전체 세트 수집해줘` | 미수집 세트 순차 수집 |
| `카드 정규화 맵 업데이트해줘` | `normalize_cards.py` canonical 맵 재빌드 |

**수집 파이프라인**
1. Playwright → shadowverse-evolve.com 클래스별 필터 수집
2. `data/carddb_json/{세트코드}.json`, `data/carddb/{세트코드}.csv` 저장

---

## 📊 통계 분석

| 명령 | 동작 |
|------|------|
| `클래스 점유율 분석해줘` | 전체 기간 클래스별 우승·TOP8 집계 |
| `{기간} 메타 분석해줘` | 예: `2월 메타 분석해줘`, `최근 2주 메타 분석해줘` |
| `덱 클러스터링 해줘` | 유사도 분석 → `data/analysis/deck_clusters.csv` 생성 |
| `카드 채용률 분석해줘` | 전 클래스 TOP8 덱 카드별 채용률 집계 |
| `{클래스} 아키타입 분석해줘` | 예: `エルフ 아키타입 분석해줘` |

**분석 파이프라인 (Codex MCP)**
- 입력: `data/개인전_v2.txt` (canonical 카드 코드)
- 출력: `data/analysis/` 하위 CSV + 차트

## 📝 메타 리포트

| 명령 | 동작 |
|------|------|
| `메타 리포트 작성해줘` | 전체 파이프라인 실행 → 한국어 리포트 생성 |
| `리포트 테스트해줘` | 보유 데이터 일부로 빠른 테스트 |
| `{날짜} 리포트 작성해줘` | 특정 날짜 기준 리포트 |

**리포트 파이프라인**
1. `rag/pipeline.py` → `enriched_data.json` + `gemini_prompt.txt`
2. Gemini MCP → 한국어 메타 리포트
3. `reports/meta_report_{날짜}.md` 저장
